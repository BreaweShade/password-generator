# Генератор паролей с оценкой сложности на основе машинного обучения

Десктопное приложение на Python с графическим интерфейсом (Tkinter) для генерации безопасных паролей и их интеллектуальной оценки с использованием модели машинного обучения Random Forest.


## Особенности

- **Генерация паролей** с настраиваемыми параметрами:
  - Длина (от 6 до 30 символов)
  - Использование цифр, спецсимволов, заглавных и строчных букв
  - Исключение неоднозначных символов (`il1Lo0O`)
- **Оценка надёжности** на основе ML-модели (Random Forest) с использованием 9 числовых признаков
- **Шифрованное сохранение** паролей в историю (Fernet, симметричное шифрование)
- **Графический интерфейс** на Tkinter с цветовой индикацией надёжности
- **Автоматические тесты** (unittest) для проверки всех ключевых функций
- **Визуализация важности признаков** модели


##  Установка и запуск

### 1. Клонируйте репозиторий

git clone https://github.com/BreaweShade/password-generator.git

cd password-generator

### 2. Установите зависимости

pip install -r requirements.txt

### 3. Запустите приложение

python password_generator.py

При первом запуске программа автоматически обучит модель Random Forest на синтетическом датасете (10 000 примеров). Это займёт 10–20 секунд. После обучения откроется графическое окно.

## Для улучшения качества модели (опционально)
Если вы хотите, чтобы модель обучалась на реальных паролях из утечек (это повышает точность и реалистичность оценки):

Скачайте файл rockyou.txt по ссылке:
https://weakpass.com/wordlists/rockyou.txt

Поместите его в корневую папку проекта (рядом с password_generator.py).
Удалите старый файл модели password_strength_model.pkl (если он есть).
Запустите программу — модель переобучится с учётом реальных данных.

⚠️ Важно: rockyou.txt не включён в репозиторий, так как его размер превышает лимиты GitHub. При необходимости вы можете скачать его отдельно.

## Запуск тестов
Для проверки работоспособности всех компонентов выполните:

python -m unittest test_password_generator.py

Ожидаемый результат: 10 из 12 тестов успешно пройдены (два расхождения связаны с калибровкой модели и не являются критическими).

## Визуализация важности признаков
Чтобы построить график важности признаков модели:

python plot_feature_importance.py

График сохранится в файл feature_importance.png и откроется на экране.

Файлы, создаваемые автоматически (не отслеживаются Git):
